# CS-300-Analysis-and-Design

CS 300 Reflection

1. What was the problem you were solving in the projects for this course?

Throughout this course, the main problem I was solving was how to efficiently store, organize, search, and display data using different data structures. Each project focused on a different way of managing data, including vectors, linked lists, hash tables, and binary search trees. The final project brought these concepts together by creating a course planner that could load course information from a file, sort courses alphanumerically, and display prerequisite information for individual courses.


2. How did you approach the problem? Consider why data structures are important to understand?

My approach to solving these problems was to first understand the requirements and then determine which data structure would be most effective. As I progressed through the course, I learned that choosing the correct data structure can significantly impact a program's efficiency and performance. For example, vectors were simple to use but required more time for searching large datasets, while hash tables provided much faster lookups. Binary search trees offered a balance by allowing efficient searching while also maintaining sorted data. Understanding these differences helped me make more informed design decisions.


3. How did you overcome any roadblocks you encountered while going through the activities or project?

One of the biggest roadblocks I encountered was working with file input, debugging compile errors, and ensuring data was loaded correctly from CSV files. I also faced challenges configuring projects in Visual Studio and managing file paths. I overcame these obstacles by testing smaller sections of code, reviewing error messages carefully, and comparing my results to the project requirements. Breaking large problems into smaller tasks made troubleshooting much more manageable.


4. How has your work on this project expanded your approach to designing software and developing programs?

This project expanded my approach to software design by showing me the importance of planning before writing code. Creating pseudocode and analyzing runtime performance helped me think more critically about how a solution would work before implementing it. I learned that successful software development involves more than simply making a program work; it also requires considering efficiency, readability, and future maintenance.


5. How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

My work in this course has changed the way I write programs by encouraging me to create cleaner, more organized, and more adaptable code. I now place greater emphasis on using meaningful function names, separating tasks into smaller functions, and selecting appropriate data structures based on program requirements. These practices improve maintainability, make debugging easier, and allow programs to be modified more efficiently as requirements change. The knowledge gained from this course has strengthened my understanding of data structures and algorithms and has provided a solid foundation for future software development projects.
